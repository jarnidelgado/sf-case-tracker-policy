Privacy Policy

SF Case Tracker – Last updated: December 2025

1. Overview

SF Case Tracker is a Chrome extension designed for internal organizational use to help agents log Salesforce case activity into a Google Sheets spreadsheet owned and managed by the organization.

We take privacy extremely seriously. This extension does not collect, sell, share, or store personal data from users or from website visitors.

2. Information the extension accesses

2.1. Your Google account basic profile (email and name)

This is obtained exclusively through Chrome’s chrome.identity API for authenticating your Google Sheets access. It enables the extension to append case logs to a shared Google Sheet using your own Google account.

The extension does not access your Gmail, contacts, files, or any other Google information.

2.2. Salesforce case fields visible on the page

To assist with auto-filling, the extension reads the following fields:

  • Case Number
  • Contact Reason

These fields are already visible on Salesforce’s interface and are not transmitted anywhere except to the Google Sheet where the agent logs the case manually.

3. What the extension does NOT collect

The extension does not collect or store any of the following:

  • Browsing history
  • Personal communications
  • User activity (keystrokes, clicks, scrolling)
  • Credentials or passwords
  • Location information
  • Sensitive personal data
  • Salesforce customer/user personal information

4. How data is used

The extension sends logged case data directly to a Google Sheet that is owned and controlled by the user's organization. No other destinations are used.

Local Chrome storage is used only for temporary functional data such as counters and preferences.

5. Data Sharing

SF Case Tracker does not sell, share, or transfer user data to any third parties. All logged data stays entirely within the organization’s Google Workspace environment.

6. Permissions Justification

This extension uses the following permissions strictly for its functionality:

  • identity – required to authenticate with Google Sheets using chrome.identity
  • storage – used to store local settings and counters
  • scripting – required to read visible Salesforce fields
  • host permissions – required to call Google Sheets API and access Salesforce UI
